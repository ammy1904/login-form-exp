# Experiment - Login Form with React State Management

## Aim
To create a secure login form using React state management for client-side validation.

## Course Outcomes Mapped
- CO2: Develop interactive and responsive front-end applications using HTML, CSS, JavaScript, and React.js.
- CO4: Debug, test, and optimize full-stack applications by analyzing performance and security aspects.

## Objectives
1. Design responsive login UI
2. Implement form validation
3. Manage user state with React hooks
4. Handle form submission
5. Display authentication feedback

## Technologies Used
- React.js
- React Hook Form
- Material UI
- JavaScript
- CSS

## Features
- Responsive login form
- Email and password validation
- Loading spinner on submit
- Success and error alerts
- Password show/hide option
- Controlled form inputs

## Learning Outcomes
- Learned how to build a responsive login form using React.
- Understood the use of React hooks for state management.
- Implemented client-side form validation using React Hook Form.
- Used Material UI components for modern UI design.
- Handled authentication feedback with alerts and loading indicators.

## Expected Output
- Validated login form
- Loading spinner on submit
- Success and error alerts
- Controlled inputs
